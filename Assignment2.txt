1.Ans-Git is a distributed version control system (DVCS) that is widely used for tracking changes in source code during software development. Developed by Linus Torvalds in 2005, Git has become the standard for version control and is used by developers and teams to manage and collaborate on software projects.

2.Ans-A Version Control System (VCS) is a software tool or system that helps manage changes to source code, documents, or any set of files over time. It allows multiple contributors to work collaboratively on a project by keeping track of different versions of files, recording changes made by each contributor, and facilitating the coordination of these changes. The primary goals of a version control system are to maintain a history of changes, enable collaboration, and provide mechanisms for tracking and reverting to previous states of a project.

7.Ans-A Git repository, often simply referred to as a "repo," is a data structure that stores metadata and a collection of files and directories associated with a project, along with the entire history of changes made to those files. In other words, a Git repository contains the source code, configuration files, and any other relevant content for a particular software project.
